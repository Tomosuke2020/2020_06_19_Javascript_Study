3.1.2 オブジェクトを利用するための準備 -new 演算子 -

オブジェクト志向の世界では、後述するような例外をのぞいて、もともと用意されたオブジェクトを直接利用することを認めていません。 オブジェクトは「自分自身の中でデータをほじできる」という性質を持っているからです。

オブジェクト
プロパティA（アプリx発動5に設定）
プロパティB（アプリy発動10に設定）
プロパティC（アプリx,yが15で衝突）

当然、データは互いに値をとるため、アプリは正しく動作しません。 そこで、オリジナルのオブジェクトには手を加えず、「オリジナルを複製したコピー」を操作することで、
データの競合を防ぐようになっているのです。

一つのデータを保持するオブジェクト

そこでオリジナルのオブジェクトに手を加えず
「オリジナルを複製したコピー」を操作することでデータの競合を防ぐようになっているのです。

オブジェクトX プロパティA プロパティB インスタンス化＝オブジェクトのコピーを生成 インスタンスα プロパティ/A ←アプリX プロパティ/B インスタンスβ プロパティ/A ←アプリy プロパティ/B それぞれオブジェクトとして別々にするため操作をする必要がないようにできる

new演算子の構文

var 変数名 = new オブジェクト名（[引数,...]）

オブジェクトには、オブジェクトを初期化するために、オブジェクトと同名のメソッドが用意されています。
この初期化メソッドのことをコンストラクターと呼びます。オブジェクト名は、正確にはコンストラクター名です。
生成されたインスタンスは変数に格納され、以降は、その変数をオブジェクトとして扱えるようになります。
インスタンス変数から格納された変数は、インスタンス変数、オブジェクト変数と呼ぶ場合もあります。

プロパティ／メソッドの呼び出し

変数名.プロパティ名[= 設定値]; 変数名.メソッド名([引数[,...]]);
