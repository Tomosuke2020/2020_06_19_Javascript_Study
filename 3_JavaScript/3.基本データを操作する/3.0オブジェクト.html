オブジェクトとは

JavaScriptのオブジェクトは名前をキーにアクセスできる配列、
連想配列（ハッシュ）と解釈しました。

オブジェクトは、単に名前がついた入れ物の集合ではなく、
オブジェクト自体が一つのものであり、
このものの特性や動作を表すために存在します。

連想配列：複数要素の集合体

「配列」data [X0001] [X002] [Z0099] [U1005] 山田 掛谷 薄井 川村

オブジェクト：一つのものを表現するために、
複数の属性情報を持つ

[mame] [seibetsu] [birth] [addres] 山田 男 1970/06/25 千葉県

JavaScriptで構文的には連想配列もオブジェクトも同じ名前つきの配列。
しかし、言葉として使い分ける場合、それぞれの意味合いは別のもの

3.1.1オブジェクト=プロパティ＋メソッド

オブジェクトはプロパティとメソッドとから構成されます。
プロパティとは「オブジェクト（モノ）の状態や特性を表すための情報」のことです。
例えば、入力フォームを表すFormのようなオブジェクトであれば、フォームの名前、フォームに含まれるテキストボックスや選択ボックス要素、フォームによる送信先などが、プロパティに相当します。

対して、メソッドはオブジェクト（モノ）を操作するための道具です。
Formオブジェクトであれば、「フォームの情報をサーバーに送信する」
「フォームの内容をクリアするなどの機能がメソッドに相当します。」

-オブジェクト----------------------------------------------------------
| フォームを表すFromオブジェクト
| 『プロパティ』 ⇄ 『メソッド』 |
| (モノの状態や特性を表す情報) ⇄ （モノを操作するための道具） |
| ・フォームの名前 ⇄ ・フォームを送信する |
| ・フォームの送信先URL ⇄ ・フォームをクリアする |
______________________________________________________________________

プロパティ／メソッドという視点から見た時、オブジェクトは「データを操作するためのさまざまな機能を持った」高機能な入れ物、と言えます。
